# Mid-day Check-in

Lightweight mid-day reset: assess what's done, what's left, and produce a clean reprioritized schedule for the remainder of the day. Reads only the current daily note and 3 prior days (not 7).

## Step 1: Gather Context (all in parallel)

1. **Google Calendar** — Run this Bash command to fetch today's events across all three calendars:
   ```bash
   TODAY=$(date +%Y-%m-%d); TOMORROW=$(date -d tomorrow +%Y-%m-%d); for cal in "primary" "family14670976850356916614@group.calendar.google.com" "20394623f738d4c89a7d45b1b05eae121f0311ca372cddab776fb78a0b308c63@group.calendar.google.com"; do gws calendar events list --params "{\"calendarId\":\"$cal\",\"timeMin\":\"${TODAY}T00:00:00Z\",\"timeMax\":\"${TOMORROW}T00:00:00Z\",\"singleEvents\":true,\"orderBy\":\"startTime\"}"; done
   ```
   **Outlook** — `mcp__Outlook__get_todays_events`
2. **Today's daily note** — `00 - Timestamps/YYYY/MM-MonthName/YYYY-MM-DD-DayName.md`
3. **IMPORTANT** **Today's modified notes** — Find and read all notes modified today using this Bash command (use a heredoc to prevent bash from mangling PowerShell's `$_`):
   ```bash
   powershell -File - << 'PSEOF'
   Get-ChildItem -Path 'D:\My Drive\DriveSyncFiles\Obsidian Vault' -Recurse -File | Where-Object { $_.LastWriteTime -ge 'YYYY-MM-DD 00:00:00' -and $_.LastWriteTime -lt 'YYYY-MM-DD+1 00:00:00' } | Select-Object FullName, LastWriteTime | Sort-Object LastWriteTime
   PSEOF
   ```
   Replace `YYYY-MM-DD` with today's date and `YYYY-MM-DD+1` with tomorrow's date. Then read all returned files, skipping: `.trash/`, Japanese vocab notes (`02 - Projects/Japanese/単語`), and the daily notes already being read in steps 3–5.
4. **Prior 3 daily notes** — Use `mcp__obsidian-rag__get_recent_daily_notes(days=3)`. Returns full note content — no path construction needed.
5. **CLAUDE.md** — Use `mcp__obsidian-rag__get_note("03 - Resources/CLAUDE.md")` — for role, rules, and constraints. Reference before producing output.
6. **PROGRESS.md** — Use `mcp__obsidian-rag__get_note("03 - Resources/PROGRESS.md")`

No need to read Recurring Tasks unless a specific issue warrants it.

## Step 2: Extract and Analyze

### From today's daily note
- **Done:** completed time blocks and tasks (`- [x]` items in Time Blocking and Tasklist sections)
- **Remaining:** incomplete time blocks (`- [ ]` items in Time Blocking)
- **Open tasks:** incomplete items in the Tasklist section (non-time-block tasks)

### From prior 3 daily notes
- **Carry-forwards:** any `- [ ]` tasks from prior days with no future due date that haven't appeared yet today
- **Patterns:** anything notable (recurring incomplete items, energy dips at certain times)

### From calendars
- Filter to events that haven't happened yet (start time > now)
- These are fixed — do not replace them with time blocks (Rule 6)

### Current time
Infer approximate time from context (e.g., user says "just got back from lunch", time blocks already checked off). If unclear, note the ambiguity and assume mid-afternoon.

## Step 3: Output

Produce a focused briefing. Keep it tight — this is a quick reset, not a full report.

---

# 🔄 Mid-day Check-in — [Weekday], [Month] [D]

## ✅ Progress So Far
[Bulleted list of completed time blocks and tasks. If light on completions, acknowledge it neutrally — no judgment.]

## 📅 Remaining Calendar
[Events still to come today, sorted by time. Format: `HH:MM – HH:MM — Event Name`. If none, say "Nothing else scheduled."]

## ⏰ Reprioritized Schedule
[Rebuild the remainder of the day as time blocks. Rules:
- Start from now (skip time already passed)
- Preserve any remaining calendar events as anchors — do NOT create time blocks for them
- Apply weekly UOA/Billingsley balance (scan this week's blocks from prior notes)
- Take time from work blocks before moving anything personal (Rule 1)
- Weave in any carry-forward tasks that are still actionable today
- Format: `HH:MM – HH:MM — [Block Name] #tag`]

## ➡️ Pushing to Tomorrow
[List anything from today's original plan that clearly won't happen. Be decisive — better to commit to tomorrow than leave it floating. If nothing needs to move, say "Nothing — plan is achievable."]

## 🧭 Advisor

[This section is generated by the advisor agent. See Step 2b.]

**Synthesized tasks**
[Use `mcp__obsidian-rag__search_vault` with a query based on today's dominant theme or project (e.g., if today is heavy on Billingsley, search "billingsley tasks todo"). Return 2–4 suggestions from across the vault that would be genuinely useful to tackle today given the current context. These are not existing tasks — they are surfaced ideas from notes, resources, or project docs.]

**Modified files**
[Bullet list of all files modified today.]

## Step 2b: Dispatch Advisor Agent

After producing the briefing up through the Reprioritized Schedule section, dispatch the **advisor agent** using the Agent tool:

```
Agent(
  subagent_type: "advisor",
  description: "Midday guidance",
  prompt: "<provide the advisor with: what's been completed so far today, what's remaining, carry-forwards, the reprioritized schedule, any items being pushed to tomorrow, and energy/mood context if available. Then ask: 'Given what's done and what's left, what's the single most important thing Charles should focus on right now, and is there anything he should watch out for this afternoon?'>"
)
```

Insert the advisor's response into the 🧭 Advisor section of the briefing. Do not editorialize or rewrite the advisor's output — present it as-is.
